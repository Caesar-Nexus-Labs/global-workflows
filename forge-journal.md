# Technical Journaling (/forge-journal)

## 1. Description
Records the "Why" behind technical decisions and keeps a continuous log of project evolution for future context recovery.

## 2. Execution Logic
1. **Decision Capture**: Document Architectural Decision Records (ADR) during planning.
2. **Daily Logging**: Record accomplishments, failed experiments, and unexpected breakthroughs.
3. **Learning Synthesis**: Extract reusable patterns or "gotchas" for the Antigravity Brain.
4. **Checkpoint Sync**: Cross-reference with `/checkpoint-create` for historical traceability.

## 3. Iron Laws
- **Continuous Registry**: If a major decision is made, it *must* be logged in the journal.
- **Context Preservation**: Journals must include enough context for a fresh agent to understand the rationale.
- **ISO Standard**: All entries must be in clear, technical English.
