# Strategic Logic Deconstruction (/market-scout-logic)

## 1. Description
A high-fidelity analysis framework used to dismantle complex product architectures and strategic frameworks. It identifies the hidden "Logic DNA" that drives successful systems, allowing for ethical reconstruction and adaptation within the Caesar Nexus ecosystem.

## 2. Caesar Nexus Execution Loop

### Phase 1: Surface Mapping (Scout)
- **Action**: Map all entry points, public APIs, and user interface hierarchies.
- **Agent**: `Logic Deconstructor Agent` execution.
- **Check**: Identify core technologies and structural dependencies.

### Phase 2: Structural Dismantling (Blueprint)
- **Action**: Use AST-inspired analysis (from `white-label-ast-transform`) to map deep logic flows.
- **Check**: Define the relationship between frontend triggers and backend state transitions.

### Phase 3: Logic Extraction (Cook)
- **Action**: Isolate unique algorithms, data structures, and orchestration patterns.
- **Verify**: Ensure zero brand-specific metadata is retained.

### Phase 4: Compatibility Audit (Gate)
- **Action**: Verify the extracted logic against Caesar Nexus "Iron Laws".
- **Call**: `/nexus-audit` for brand and security integrity.

### Phase 5: Implementation Handoff (Watch)
- **Action**: Document the "Clean Room" version of the logic.
- **Move**: Handoff to `/forge-plan` for creative integration into new Caesar products.

## 3. Iron Laws
- **Commercial Safe**: Terms like "Reverse Engineer" or proprietary brand names are forbidden in final documentation.
- **Clean Room Protocol**: Final outputs must be 100% de-branded and independent of source origin.
- **Value Focus**: Extraction must focus on functional value, not literal reproduction.
