# Knowledge Graph Analysis (/forge-gkg)

## Caesar Nexus Execution Logic

### Phase 1: Indexing & Setup
- **Action**: Generate or refresh project's symbol index/GKG.

### Phase 2: Visualization & Patterns
- **Action**: Identify circular dependencies and high-complexity clusters.

### Phase 3: Gap Discovery
- **Action**: Find undocumented modules or inconsistent patterns.

### Phase 4: Strategy Mapping
- **Action**: Identify cross-module impact areas.

### Phase 5: Handoff to Planning
- **Call**: `/forge-plan`
- **Action**: Feed GKG insights into strategic transformation plans.

## Iron Laws
- **Accuracy**: GKG must represent actual code on disk.
- **Privacy**: No repository metadata shared with external untrusted agents.
- **Contextual Truth**: GKG findings are primary input for Brownfield projects.
